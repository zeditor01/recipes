IBM's Z Data in Motion Options

The three primary options for replicating Z data to other platforms are Q Replication, CDC Replication and Datagate. They all have different characteristics that suit them to different scenarios.

The notes below provide some broad positioning statements. Click on the appropriate button to reveal a range of implementation worked examples for that replication technology

CDC Replication is aimed at heterogenous data replication.

  • It supports all the regular relational databases ( Db2 Oracle, SQL Server etc... ) as sources and targets.
  • Additionally, it supports non-relational databases such as IMS and VSAM.
  • And it supports non-relational targets such as Apache Kafka (various distributions of Kafka).

CDC Capture Agents stream changes to CDC Apply agents over TCPIP sockets. Devops management is supported with both graphical and scripting interfaces.

Q Replication is primarily aimed at homogeneous DB2 Replication. (both Db2 z/OS and DB2 LUW). If your sources and targets are all Db2, then Q Replication offers some attractive benefits such as:

  • It supports in-line DDL Replication between Db2 z/OS sources and Db2 z/OS targets..
  • It supports publishing to MQ in open formats, which avoids the need to co-ordinate management Capture, Apply and the network in-between.
  • It is the basis for the GDPS Active-Active continuous availability solution.

Q Capture Agents stream changes to Q Apply agents over Websphere MQ messaging channels. Devops management is supported with both graphical and scripting interfaces.

Datagate is an ultra-efficient data replication protocol between a Db2 z/OS V12 source and a Db2 Linux target via Cloudpak for Data on a number of Redhat openshift platforms.

  • It supports a choice of conventional row-based Db2 targets, or columnar Db2 targets that are optimised for Analytics.
  • It provides the option to accelerate analytics SQL queries on Db2 z/OS by transparently routing them through to a columnar copy on Db2 Linux.
  • It provides the ability to place your operational Db2 z/OS data in the cloud to support new cloud-native development projects.

Datagate is a much narrower use case that the more generalised CDC and Q Replication tools. But if your requirements fit within that envelope it is the best replication option by far (z/OS software consumption charges, cpu cycles, low data latency, devops controls for changes, etc... ).
