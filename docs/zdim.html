Building a Data Fabric for Mainframe Data

  • Z Data in Motion
  • Z Data at REST
  • ERIS

The diagram below depicts a very common approach for building a Data Fabric that contains data from IBM Z Systems of Record. It is to move and replicate large amounts of data to external Digital Integration Hubs whose purpose is curate data to serve new cloud-based application services.

Z Data in Motion

For customers that adopt a data fabric based on this pattern, you will need low latency data replication and event publishing solutions. This section discusses the primary solutions that IBM offers for capturing and consuming "Z Data in Motion" whichever Data Fabric architecture is preferred.

The Z Data at REST and ERIS sections describe alternative approaches that reduce or avoid the extensive copying of data to other platforms. Please consider these approaches as complimentary or alternative approaches to building a Data Fabric for mainframe data.

CDC Replication is aimed at heterogenous data replication, and streams changed data over TCPIP sockets.

  • It supports all the regular relational databases ( Db2 Oracle, SQL Server etc... ) as sources and targets.
  • Additionally, it supports non-relational databases such as IMS and VSAM.
  • And it supports non-relational targets such as Apache Kafka (various distributions of Kafka).

Q Replication is primarily aimed at homogeneous DB2 Replication, and streams changed data over Websphere MQ links.

  • It supports in-line DDL Replication between Db2 z/OS sources and Db2 z/OS targets..
  • It supports publishing to MQ in open formats, enabling a pub/sub integration model with MQ consumers.
  • It is the basis for the GDPS Active-Active continuous availability solution.

Datagate is a lightning fast data replication protocol between a Db2 z/OS V12 source and a Db2 Software Container, such as IDAA or Datagate.

  • It supports a choice of conventional row-based Db2 targets, or columnar Db2 targets that are optimised for Analytics.
  • It supports acceleration of analytical SQL queries on Db2 z/OS by transparently routing them to a columnar Db2 Software Container.
  • It provides the ability to place your operational Db2 z/OS data in the cloud to support new cloud-native development projects.

Datagate is a much narrower use case that the more generalised CDC and Q Replication tools, because it only supports Db2 Software containers as targets. But if your requirements fit within that envelope it is the best replication option by far ( lower z/OS software consumption charges, redcued cpu cycles, ultra-low data latency, devops controls for changes, etc... ).
