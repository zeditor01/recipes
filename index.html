Worked Examples for Mainframe Data Integration Patterns.

Enterprises that run operational systems on IBM z/OS are creating and managing data that is the essential raw material for three critical corporate initiatives

  1. Continuous Transaction Processing to support the Core Business
  2. Develop AI models to help automate and govern those core business transactions
  3. Deploy realtime Data APIs to be consumed by Cloud applications as the public face of the business

Expressed another way, the challenge is to find the best way to create APIs ( for transactions, data access, and AI scoring ) based on the mainframe data, to best serve the public presence of an Enterprise.

"Zeditor" is a practical cookbook for the 3 major patterns used for accessing and exploiting data that is created and managed on IBM Mainframe Systems. It comprises documented implementation audit trails, demonstration videos, and access to hands-on test drive systems

We are all employed by IBM Australia as System Z technical specialists, and share a passion for building things on our z/OS test rigs. (we all use IBM Z Development and Test software to run z/OS systems as a virtualized guest of Linux on some beefy x64 systems at home).

Our z/OS test rigs are what we use for customer demonstrations and self education. As we work with clients to deploy z/OS software in real world environments, we apply practical experiences from the real world to enhance our demonstration platforms. As we grow our experiences, we document them for future reference, and include some of those lessons learnt that you just don't seem to get in the formal product documentation.

As our recipe notebooks for getting things done grew larger and larger, we wanted a permenant place to safely store them and share them. Github was the obvious repository for our code and documentation. This website will grow over time, so please drop by from time to time to find new recipes.
