The "Z Data at REST" pattern is motivated by avoiding the costs, complexity and timescales of moving the data off Z, and making use of the wide range of data service solutions that exist to serve data direcly from z/OS. This philosphopy behind Cloudpak for Data is to leave data where it is (as far as possible) and use a data virtualization layer to access data on demand from remote sources like z/OS.

The worked examples in this section show how z/OS data resources can be surfaced using a range of APIs (REST, SQL, ODBC, JDBC etc....) for consumption by cloud applications directly, or by Data and AI integration solutions like Cloudpak for Data.

DB2 DDF is a standard component of Db2 z/OS.

  • It can serve REST APIs based on calls to SQL and Stored Procedures.
  • It can serve a variety of SQL-based APIs ( JDBC, ODBC etc... ).
  • It is tightly integrated into the Db2 z/OS service (Reliability, Performance, Secuirty etc...)
